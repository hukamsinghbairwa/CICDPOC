View WebSocket Port Details
Screen Heading: Security > Ports > View WebSocket Port Details
Use the View WebSocket Port Details screen to view or specify the settings for an WebSocket port.
Click Return to Ports to return to the list of ports that are currently defined on the server.
Click Edit WebSocket Port Configuration to edit general configuration information about the WebSocket port that are currently defined on the server.
Click WebSocket Endpoints to view active and inactive sessions for WebSocket server endpoints. When WebSocket port is running, you can view the sessions that are currently active. To view the sessions that are currently active, click on the current number of sessions.
WebSocket Listener Configuration
Use this section to view general configuration information about this port.
Port
Specifies the number of the port to which you want to assign an WebSocket listener. Type a port number that is not used by another port on your server.
Alias
An alias for the port. An alias must include one or more of the following: ASCII characters, numbers, underscore (_), period (.), and hyphen (-).
Description (optional)
A description of the port.
Package Name
Specifies the name of the package that you want to associate with this port. When you assign a package to a port, the server enables and disables the port in conjunction with the package--if you disable the port, the server disables the port automatically. A package that you assign to a port also retains its port assignment when it is published to other servers. If you do not have a package to assign to the port, leave the Package Name setting at WmRoot (the server's core package). This makes the port available whenever the server is running.
Bind Address (optional)
Optional. IP address to which to bind this port. Specify a bind address if your machine has multiple IP addresses and you want the port to use this specific address. If you do not specify a bind address, the server picks one for you.
Backlog
The number of requests that can remain in the queue for an enabled port before Integration Server begins rejecting requests. The default is 200. The maximum value is 65535.
WebSocket Server Endpoints
This section displays the server related URI that represents the address of the service and the sessions that are currently active for this WebSocket port, however, you can access this screen if there are active session available for an endpoint.
Click Return to Ports to return to the list of ports that are currently defined on the server.
WebSocket URI
Indicates the server relative URI that the WebSocket client uses to connect to the Integration Server acting as a WebSocket server.
Active Sessions
Indicates the number of active sessions in the server for the WebSocket URI that are currently active. Click the current number of sessions, Integration Server displays the WebSocket Server Sessions screen.
WebSocket Server Sessions
This section displays the list of WebSocket server sessions that are currently active.
Click Return to Ports to return to the list of ports that are currently defined on the server.
Local Address
Displays the local address and port of the server for that particular session.
Remote Address
Displays the remote address and port of the client for that particular session.
Protocol Version
Displays the WebSocket protocol version. Integration Server supports only WebSocket protocol version 13.
Idle Timeout
Indicates the length of time in milliseconds, the WebSocket can keep an unused connection open.
Secure
Indicates if the WebSocket session is secure. Yes indicates the session is secure. No indicates the session is not secured.
Suspended
Indicates if the session is suspended. Yes indicates the session is suspended. No indicates the session is not suspended.
Use the following controls to manage server sessions. Specifically, you can:
Option
Description
Suspend
Suspend the selected WebSocket session until the user resumes or close the task.
Resume
Resume the selected WebSocket session.
Close
Close the selected WebSocket session.
Disconnect
Disconnects the selected WebSocket session.
